LAN SCANNER
Technical Specifications

Write a scanner that obtains system information about computers on a local network.

Terms and requirements:
1. The domain administrator's name/password is given as input.
2. The required list of information in appendix 1.
3. Minimizes the total number of network requests as much as possible.
4. If it is possible to minimize/compress/encrypt network traffic, we do.
Depends on the services and services of Windows itself that respond to our requests.
5. Architecturally, the scanner is a single program. In the sense that it has no response part/stagers/agents,
that would need to be downloaded over the network and run before the scan can be performed.
That is, the program only pulls the regular Windows services on the remote hosts and relies on them completely.
6. The implementation language is C++ or C#.
PowerShell/.bat scripts get whole chunks into the Windows system log, so execution as a script is not suitable.
7. Execution as a .dll 32/64 bit, according to the "Code and Assembly Formatting" document.
The entry point is compatible with Cobalt Strike loads.
The .dll size is up to 1 megabyte.
Must be able to pass startup parameters to the .dll.
8. Provide for multi-threading, and prioritize scanning.
Optimize for scanning networks of 1k nodes or more.
